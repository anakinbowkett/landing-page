Listing Outcomes

Writing down every possible result from one or two events — without missing any

What This Is

An outcome is one possible result from an event — like rolling a 4 on a dice, or getting Heads on a coin. When a question asks you to list outcomes, it wants you to write down every single possible result, in an organised way, so nothing is missed. The key skill is being systematic: you work through the possibilities in a fixed order so you never accidentally skip one or write the same one twice. On a GCSE paper you will often need to list outcomes from two events happening together — for example, flipping a coin and spinning a spinner at the same time.

Key Terms You Must Know

Outcome

A single possible result from an event. Rolling a dice has 6 outcomes: 1, 2, 3, 4, 5, and 6. Flipping a coin has 2 outcomes: Heads and Tails. Every outcome must be distinct — you cannot list the same result twice.

Event

An action that produces an outcome. Spinning a spinner is one event. Flipping a coin is another event. When two events happen together, you need to combine every outcome from the first event with every outcome from the second.

Sample Space

The complete list of all possible outcomes from an experiment. If you spin a 3-colour spinner and flip a coin, the sample space has 6 outcomes. The word "sample space" is what the examiner calls the full list — whether it is written as a list, a table, or a diagram.

Sample Space Diagram

A table that shows all possible combined outcomes from two events. One event labels the rows; the other labels the columns. Each cell in the table contains the combined outcome for that row and column. It is the neatest way to organise outcomes and guarantees none are missed.

Systematic

Working through possibilities in a fixed, organised order. For example: fix the first event at its first value, then write every outcome for the second event. Then move to the second value of the first event, and repeat. Being systematic means missing outcomes is impossible.

The Method — Step by Step
Grade 1–3 topic. You need to list all outcomes from one or two events in an organised way and count them correctly.
1

Identify each event and write out its outcomes separately

Read the question and find each separate event. Write the outcomes of the first event as a list going across, and the outcomes of the second event as a list going down. Doing this first means you know exactly how many rows and columns your table will need — it prevents the table from being the wrong size.

Count the outcomes of each event. The spinner has 3 outcomes; the coin has 2. Multiply these numbers together: 3 × 2 = 6. This tells you exactly how many combined outcomes the complete list must contain. Use this total as a check at the end — if you have a different number, you have missed one or repeated one.

Spinner (Red, Blue, Green) and Coin (Heads, Tails): 3 × 2 = 6 combined outcomes to find.
2

Draw the table — one event labels the columns, the other labels the rows

Draw a table with a blank cell in the top-left corner. Write each outcome of the first event as a column heading, and each outcome of the second event as a row heading. The blank corner cell is where the row headings and column headings meet — leave it empty. A table structure forces you to fill every cell, so a missed combination becomes physically obvious.

Table drawn with Red, Blue, Green across the top and Heads, Tails down the side. Six empty cells wait to be filled.
3

Fill each cell by combining the row heading with the column heading

Start at the top-left cell. The column heading is "Red" and the row heading is "Heads" — so write "Red, Heads" in that cell. Move right along the first row, filling every cell: "Blue, Heads" then "Green, Heads". You are keeping the coin result fixed at Heads and changing the spinner colour each time.

Move down to the second row. The row heading is now "Tails". Work across again from left to right: "Red, Tails", then "Blue, Tails", then "Green, Tails". The row heading changes; the column heading changes with each cell. Continue this pattern until every cell in the table is filled.

Table complete: R,H — B,H — G,H in the top row; R,T — B,T — G,T in the bottom row. Six cells, six outcomes. Count matches the 3 × 2 = 6 prediction.
4

Count the outcomes and answer the question

Count every cell in your completed table. This total is the number of possible outcomes. Write this number clearly beneath the table so the examiner can see it. If the question asks "how many possible outcomes are there?", this count is your answer — state it as a single number.

If the question then asks for the probability of a specific outcome — for example, "what is the probability of getting Blue, Tails?" — look for that outcome in your table, count how many times it appears (it should be exactly once for a fair experiment), and write the probability as that count over the total number of outcomes. The table you have built does all the hard work for any probability question that follows.

6 outcomes in total. Blue, Tails appears exactly once in the table. P(Blue, Tails) = 1/6.
5

In the event of a single event — use a plain list instead of a table

If the question involves only one event — for example, a single spinner with sections 1, 2, 3, 4 — a table is unnecessary. Write the outcomes as a straight list, one per line, in order from smallest to largest. Number each outcome so you can count them without losing track. The same rule applies: write every possible result exactly once.

If the question involves three events — for example, three coins — the table approach cannot hold all the combinations in two dimensions. Instead use a systematic list: fix the first coin at Heads, then list every combination of the other two (HH, HT, TH, TT). Then fix the first coin at Tails and repeat. This gives 2 × 2 × 2 = 8 outcomes in a reliable order.

Three coins: fix coin 1 at H, list all 4 combinations for coins 2 and 3, then fix coin 1 at T and list again. Total: 8 outcomes.
