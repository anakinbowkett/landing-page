Frequency Tables

Record data neatly and read it back to answer questions

What This Is

A frequency table is a neat way to count how many times each item appears in a set of data. "Frequency" just means "how many times". Instead of reading through a messy list of results, you use the table to see at a glance how many times each option was chosen — and then answer questions from it quickly and without making mistakes.

Key Terms You Must Know

Frequency

How many times something happens or appears. If 8 students chose Blue, the frequency for Blue is 8.

Tally

A quick counting method using marks. You draw one vertical line per item. Every fifth mark is drawn diagonally across the previous four — making groups of 5 easy to count.

Frequency Table

A table with (at least) two columns: one for each category and one for its frequency. Some tables also include a tally column as a middle step.

Total Frequency

The sum of all the frequencies. It must equal the total number of items in the original data set — use this as a check at the end.

Mode

The item that appears most often — the one with the highest frequency. In a frequency table, look for the largest number in the frequency column, then read left to find its category name.

The Method — Step by Step
Grade 1–3 topic. Every question gives you the raw data and an empty (or partly filled) table. Your job is to count carefully, fill in the table correctly, and read the right value back out. Take it one column at a time — never rush the counting step.
1

Read the table — understand what you are being asked to count

Look at the left-hand column of the table. Each row names one category — these are the options you are counting. Read every category name before you do anything else so you know exactly what goes in each row.

Count the number of rows to know how many categories there are. This tells you how many different groups you need to count through — so you do not accidentally skip one.

e.g. The table has four rows: Blue, Red, Green, Yellow. You need a frequency for each one.
2

Go through the raw data and tally — one mark for each item

Start at the very first item in the data list and draw one tally mark in the correct row. Cross that item off or tick it so you know you have counted it. Never count from memory — always work through the list item by item.

When you reach your fifth mark for any row, draw it diagonally across the previous four. This bundles your marks into groups of 5, making them far quicker to count at the end.

e.g. Blue chosen 8 times → one complete gate (5) plus 3 more marks: |||| |||
3

Count the tallies and write the frequency number in the final column

Count each group of five as "5", then count the leftover marks separately and add them on. Write the total as a single number in the Frequency column on the same row. Do this for every row before moving on.

Count a full "gate" (the crossed group) as 5 straightaway — do not count the individual lines inside it. Then count any loose marks after it and add them on. So |||| ||| is 5 + 3 = 8, not 1 + 1 + 1 + 1 + 1 + 1 + 1 + 1.

e.g. Blue: one gate + 3 marks = 5 + 3 = 8. Write 8 in the Frequency column on the Blue row.
4

Add all the frequencies to find the total — and use it to check your work

Add together every number in the Frequency column. Write the sum in the Total row at the bottom of the table. This total tells you how many items were in the entire data set.

Compare your total to the number of items the question says were in the survey or experiment. If they match, your table is correct. If they do not match, you have missed or double-counted an item — go back through your tallies.

e.g. 8 + 5 + 4 + 3 = 20. The question says 20 students were asked — the totals match, so the table is right.
5

Find the mode — look for the biggest number in the Frequency column, then read left for its name

Run your eye down the Frequency column and find the largest number. That number tells you which category was chosen most often. Circle it so it stands out clearly on your page.

Now read left from the circled number to get the category name in the first column. The mode is always the category name — not the frequency number itself. Write "Mode = Blue", not "Mode = 8".

e.g. Biggest frequency = 8. Read left → Blue. Mode = Blue.
6

In the event of a grouped frequency table — read each class interval carefully

A grouped frequency table uses intervals instead of single values — for example "10–19" or "20 ≤ x < 30". Each data value goes into exactly one interval, so read the boundaries carefully before you tally.

The symbol ≤ means "included in this group". The symbol < means "not included — it goes in the next group up". So a value of exactly 20 goes in the 20 ≤ age < 30 group, not the 10 ≤ age < 20 group.

e.g. If someone is 20 years old, they go in "20 ≤ age < 30" — not "10 ≤ age < 20", even though 20 is close to that group's top boundary.
